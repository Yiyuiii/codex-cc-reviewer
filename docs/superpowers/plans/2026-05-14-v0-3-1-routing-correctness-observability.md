# v0.3.1 Routing Correctness And Observability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a focused `0.3.1` maintenance release that prevents silent routed-diff evidence loss, makes parser fallback visible, and aligns docs/release validation with the `0.3.0` routing behavior.

**Implementation status:** Complete. Final diff review accepted one cleanup and three low-risk usability/observability improvements: remove duplicated diagnostics budgeting, use a non-tag-shaped fallback sentinel, render fallback line counts as `n/a`, and add diagnostics when only some `diff --git` blocks parse. Later convergence passes also added defensive manifest escaping, all-dropped block-count diagnostics, diagnostics weight precomputation so clean diffs do not reserve unused diagnostics budget, `text` fences for raw fallback evidence, and small readability cleanup in packet preparation.

**Architecture:** Keep the one-tool `cc_review` contract and the supported `claude -p` backend. Add a packet-level fallback for non-empty diffs that cannot be parsed into `diff --git` blocks, expose structural diagnostics without echoing raw diff content, and update release/docs evidence so the routed `review_diff` path is validated after restart. Do not add public routing customization, filename denylisting, preview JSON, or background-mode product support.

**Tech Stack:** TypeScript, Zod, Commander, Vitest, Node.js 20+, MCP TypeScript SDK.

## Accepted Plan Review Decisions

- Use a structured return from `prepareGitDiffBlock`: `{ markdown, diagnostics }`. This keeps parsed diff ownership local to packet construction, parses the redacted diff exactly once, and lets `prepareVariableBlocks` merge fallback diagnostics into the existing packet diagnostics.
- Add a single raw fallback formatter in `src/review/context-router.ts` rather than hand-writing a second markdown shape in `packet.ts`.
- Trigger fallback only when `redacted.trim().length > 0 && parsed.length === 0`. Empty diffs keep the current no-body behavior.
- Use `risk: unparseable; diff_parse_failed; raw_fallback` as the manifest/evidence reason.
- Compute fallback inclusion from actual truncation: `full` when the raw fallback was not truncated, `partial` when it was truncated.
- Use `[unparsed-diff]` as the fallback sentinel and render its manifest change count as `n/a`.
- Emit a diagnostic when parsing succeeds for some but not all `diff --git` blocks: raw diff evidence may be incomplete.
- Keep the release validation gate as a documented maintainer requirement in `AGENTS.md` for routing/evidence headline changes. Do not add live-Claude checks to CI.
- Prepare `0.3.1-rc.0` metadata only after implementation tests are green. Stable `.release-validation/v0.3.1.md` is not generated by this implementation pass.
